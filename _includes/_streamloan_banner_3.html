Design Iterations

Developing the loan process flow

According to user interviews and industry research, loan officers need help accomplishing several key tasks:

  1. See New Requirements from the underwriter
  2. Assign Loan Requirements to the borrower
  3. Accept Or Decline Documents uploaded by the borrower
  4. Email Accepted Documents to the underwriter for approval
Iteration 1: Deciding on navigation tabs

Early iterations translated key tasks that a loan officer needs to complete into submenu categories.

I chose navigation tabs since submenu categories needed to be separated into distinct sections and tabs provide a clear indication of where the user is on the app.

Iteration 2: Incorporating existing design patterns and visual cues

Segmented controls and check-mark lists were added to the designs below. The reason I defaulted to the complex architecture was:

  1. It was an existing pattern the Streamloan user was familiar with
  2. It was easier for the client to implement

Using the current design system, I added clock icons and progress bar to help a loan officer track conditions at a glance.
