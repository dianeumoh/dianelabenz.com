Dashboard Iterations

How will the dashboard address user goals?

After completing the initial application process, the user is taken to the home page via a welcome email. For Lantern's home page dashboard, I thought about the following questions:

  1. How does the user know what to do next?
  2. How many steps does it take to search and apply for jobs?
  3. Why would a user return to the Lantern platform?
Early Iterations: Too many steps to see relevant jobs

Early dashboard iterations included next step instructions for the user but it didn't address the user's ultimate goal, finding the right job.

The Jobs Dashboard: Addressing the why

The number one reason that the user would back to Lantern is to search and apply for new opportunities. By making the home page the jobs page, the user is able to quickly accomplish the task at hand.
