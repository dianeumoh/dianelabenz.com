Dashboard Iterations

How will the dashboard address user goals?

After completing the initial application process, the user is taken to the home page via a welcome email. For Lantern's home page dashboard, I thought about the following questions:

  1. How does the user know what to do next?
  2. How many steps does it take to search and apply for jobs?
  3. Why would a user return to the Lantern platform?
Dashboard V1 & V2: Too many steps to see relevant jobs

Early dashboard iterations included next step instructions for the user but it didn't address the user's ultimate goal, finding the right job.

The only step that a user needs to complete before applying to a job is the certification step. The checklist ran the risk of being overwhelming to the user while giving the perception that there are more steps than actually required.

Dashboard V3: Addressing the why

The number one reason that the user would return to Lantern is to search and apply for new opportunities. By making the home page the jobs page, the user is able to quickly accomplish the task at hand.

Final Design Rationale

From the initial login, users can see important announcements in the New For You section and view the latest jobs.
