Discovery

Developing the search and discovery flow

In the current Countable search flow, users have to try each filter and decide which option will return the most relevant result. 5/5 users were confused by the filter term "profiles".

I concluded that a simplified search flow would better connect users with the issues.

Design Iterations

From multiple iterations to a discovery experience that engages

I looked at different search interface elements before landing on the three versions below.

Low-fi Sketches: How do political apps handle search & discovery?

I sketched designs based on common UI patterns found in political news apps and employed dot voting to settle on elements highlighted below.

Wireframes: Designing search and filter functionalities

I reimagined the Search page without the segmented controls to reduce friction.

During usability tests, users found Countable's issues hashtags (located on articles in the Feed) helped give context to news stories. I decided to surface the hashtags on the search page for added issue discoverability.

Article categories were added to the Feed to reduce cognitive load for the user.

High-Fi: Implementing the navigation bar and finalizing the flow

The navigation bar was added to the Feed after realizing my initial design would introduce a new user experience potentially causing friction for current users.
